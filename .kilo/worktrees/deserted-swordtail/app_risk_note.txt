# App Security Vulnerabilities and Risks Report
Generated: 2026-05-06

## Critical Issues

### 1. Missing Shop Context Authorization in APIs (OfflineSyncController.php)
- **Location**: app/Http/Controllers/Api/OfflineSyncController.php
- **Issue**: `syncSales` and `syncInvoices` endpoints don't verify products belong to the authenticated user's shop
- **Impact**: Any authenticated user can sync sales/invoices for products from other shops
- **Fix**: Add shop_id validation for each product before processing

### 2. Missing Shop Filtering in Supervisor RefundController
- **Location**: app/Http/Controllers/Supervisor/RefundController.php (lines 19-32)
- **Issue**: `index()` shows ALL pending refunds across all shops
- **Impact**: Supervisor can view/approve/reject refunds from other shops, exposing sensitive data
- **Fix**: Add `->where('shop_id', auth()->user()->shop_id)` to queries

### 3. Cross-Shop Till Access
- **Location**: app/Http/Controllers/Cashier/TillSessionController.php (lines 141-158)
- **Issue**: `approve()` allows any supervisor/manager/owner to approve ANY till session, regardless of shop
- **Impact**: Users can approve till sessions from other shops
- **Fix**: Add `->where('shop_id', auth()->user()->shop_id)` to till query

## High Issues

### 4. Hardcoded Routes in Views
- **Location**: resources/views/owner/sell.blade.php (line 329)
- **Issue**: Receipt URL hardcoded as `/owner/receipt/` instead of using named route
- **Impact**: Will break if route changes; not portable across roles
- **Fix**: Replace with `{{ route('owner.receipt', ['sale' => lastSale.sale_id]) }}`

### 5. Development File in Production
- **Location**: verify_sell.php (project root)
- **Issue**: PHP testing utility left in project root
- **Impact**: Information disclosure, unnecessary file exposure
- **Fix**: Delete file from production

### 6. Date Manipulation via User Input
- **Location**: InvoiceController.php (lines 102-108), OfflineSyncController.php (line 114)
- **Issue**: Allows arbitrary `created_at` values from user input
- **Impact**: Enables date-based fraud, invoice number manipulation
- **Fix**: Remove `created_at` from fillable/user input, use server time only

## Medium Issues

### 7. Potential XSS in Thermal Receipt View
- **Location**: resources/views/partials/thermal-receipt.blade.php (line 227)
- **Issue**: Product names processed with regex but not sanitized before display
- **Impact**: Potential XSS if product names contain malicious content
- **Fix**: Use `{!! Str::limit(e($desc)) !!}` or similar sanitization

### 8. Missing Rate Limiting
- **Location**: All authentication endpoints
- **Issue**: No rate limiting on auth or API endpoints
- **Impact**: Vulnerable to brute force attacks
- **Fix**: Add rate limiting middleware to auth routes

### 9. Credit Sale Validation Gaps
- **Location**: app/Http/Controllers/POS/SaleController.php (lines 146-154)
- **Issue**: Customer phone uniqueness not enforced before creating debt record
- **Impact**: Duplicate customer records, potential data inconsistency
- **Fix**: Validate unique constraint before Customer::firstOrCreate

### 10. Exception Exposure
- **Location**: app/Http/Controllers/POS/SaleController.php (line 203)
- **Issue**: Returns raw exception message to client: `'message' => $e->getMessage()`
- **Impact**: Information disclosure, potential leak of sensitive system details
- **Fix**: Return generic error message, log actual exception
